# 采访脚本 · 12 项决策

在动手做 playground 前，用 3 组 AskUserQuestion（每组 4 问）锁定这 12 项决策。**即使用户说"你定就行"也要走完**——后面 4 轮 playground 都基于这些答案，采访跳了就会返工。

每项有 3 个选项 + 1 个 **Recommended**（默认项）。用户答"全推荐"就按 Recommended 一路走。

---

## 采访前：品牌资产检查

先问一句：**"有自己的 logo / 主色 / slogan 吗？上传一下我看看。没有的话用默认（AI Spark · 火苗 logo · #1E40FF · 「始于火花·成于实战」）。"**

如果用户给了图片，用 Read 工具读图，然后在采访前插一个 **风格冲突仲裁问题**：

> 你给的主色饱和度比较高，但 "course-deck" 默认是克制 Keynote 风（白底 + 品牌色 ≤10% 点缀）。两种处理方式：
> - **A · 克制优先**：用你的主色，但只做点缀（按钮、重点词、下划线），整体仍然白底。
> - **B · 品牌优先**：让主色在封面、分隔页更强势（大色块），可能偏离克制风。
> - **Recommended**：A · 克制优先（这套设计系统的底色就是克制）

锁定答案后进入三组。

---

## 第一组 · 基础架构（4 问）

### 1. 技术框架

**问**：PPT 技术栈？

- **A · 纯 HTML + CSS + JS 单文件**：没依赖，离线也能放。
- B · reveal.js：主题多，但要引 CDN。
- C · Slidev：Markdown 写，但需要 Node 构建。
- **Recommended**：A（skill 整套设计都为单文件优化，选 B/C 要自己重写 deck controller）

### 2. 长宽比

**问**：屏幕比例？

- **A · 16:9**：标准投影、1080p、4K。
- B · 16:10：部分笔记本屏。
- C · 4:3：老投影仪。
- **Recommended**：A（除非用户明确说要给老设备放）

### 3. 动画基调

**问**：切页 / 入场动画？

- **A · 克制 Keynote 风**：淡入 + 轻微上移（translateY 14px），0.85s ease。
- B · 无动画：纯瞬切。
- C · 戏剧型：滑入、缩放、翻转。
- **Recommended**：A（**C 是明确禁用的**，skill 不支持。选 B 也可以但会丢失节奏感）

### 4. 页内步进（stepped reveal）

**问**：有"三条规则 / 三个动作"这类需要逐条出现的页吗？

- **A · 用步进**：按空格逐条显示。
- B · 一次显示：全部 fade-up 一次完。
- C · 不要这类页。
- **Recommended**：A（步进是讲师停顿消化的抓手，强烈推荐）

---

## 第二组 · 关键组件样式（4 问）

### 5. 目录树（tree）

**问**：要不要用代码风的文件树展示项目结构？

- **A · 等宽字体 + 品牌色高亮**：`articles/ draft/ final/` 这种树结构，文件夹蓝色，文件按状态（存档/可写/成品）配色。
- B · 用缩进列表：`<ul>` 代替。
- C · 不要树，改用图标 + 描述。
- **Recommended**：A（讲文件/目录结构时视觉冲击最强）

### 6. 流程图（flow）

**问**：要不要用节点 + 箭头展示流程（"导入 → 修改 → 定稿"）？

- **A · 节点逐个亮起 + 箭头流光**：配合步进，讲到哪个节点亮哪个。
- B · 全部一次亮：无步进。
- C · 不用流程图，改用有序列表。
- **Recommended**：A（最能表达"流水线"的意象）

### 7. 对比叙事（compare）

**问**：有没有"没有 X 的样子 vs 有了 X 的样子"这种二元对比？

- **A · 两态切换**：散乱文件堆 → 整齐流水线，按空格切。
- B · 左右分栏：同屏对比。
- C · 不用对比。
- **Recommended**：A（视觉冲击比说理强 10 倍；但只在确有"前/后"对比的节才用）

### 8. 代码块（code）

**问**：要展示代码 / 提示词 / 配置片段吗？

- **A · 终端窗口风**：深色底 `#1a1d24` + 红黄绿灯 + 行号 + 轻量语法色。
- B · 浅色 inline：`<code>` 普通样式。
- C · 不要代码块。
- **Recommended**：A（即便只展示一次，终端窗口比段落框更有"工程感"）

---

## 第三组 · 演讲辅助（4 问）

### 9. 演讲者备注

**问**：要讲师备注面板吗？

- **A · 侧边浮窗（按 N 切换）**：strategy + speak 分栏展示。
- B · 底部抽屉。
- C · 不要，讲师自己记。
- **Recommended**：A（单文件放映最大的痛点就是"讲稿对不上页"，侧边浮窗是 skill 核心价值之一）

### 10. 时间条

**问**：要不要显示本节计划用时 / 已用时？

- **A · 右上角小条（按 T 切换）**：节名 + 计划 min + 已用 mm:ss。
- B · 底部进度条。
- C · 不要，讲师看手表。
- **Recommended**：A（2.5h 单人讲最怕超时，小条一直在眼角可见）

### 11. 导航方式

**问**：翻页方式？

- **A · 全键盘**：→ 空格 回车 PageDown 下一页；← PageUp 上一页；Home/End 首末；F 全屏；N/T/? 功能。
- B · 鼠标点击侧边。
- C · 触摸手势。
- **Recommended**：A（讲师手必须在键盘上才能做步进，且支持遥控笔）

### 12. 交付形式

**问**：最终交付什么？

- **A · 单文件 .html**：双击浏览器打开就能放映。
- B · 打包成 zip（html + assets 分离）。
- C · 导出 PDF（丢失动画）。
- **Recommended**：A（skill 的核心约束就是单文件，选 B/C 要额外处理）

---

## 采访结束后

回用户一句：
> 采访完了，12 项决策锁定：[列出非 Recommended 的选择，或说"全推荐"]。品牌：[默认 / 用户自定义摘要]。
>
> 进 Playground 01 封面页。

然后进入阶段 3 的 4 轮 playground（封面 → 分隔 → 分隔桥接 → 内容模板 → 组件）。

---

## 反模式

- ❌ 一次 AskUserQuestion 塞 12 个问题——信息过载，用户会放弃认真答
- ❌ 跳过采访直接生成——后面 playground 没依据，用户一改就要重做
- ❌ 看到用户说"你定就行"就跳采访——Recommended 本身就是"你定就行"的答案，还是要走完让用户确认
- ❌ 给用户开放式问题（"你想要什么风格？"）——选项式 + Recommended 比开放式决策快 5 倍
